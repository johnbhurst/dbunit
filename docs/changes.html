Changes in version 1.4 from 1.3
  • Moved the Base64 class to the org.dbunit.util package because Visual Age for Java is not able to handle classes located in the default package.
  • New features:
    • 542034 - Detect ambiguous table names.
    • 542462 - Added a system property to disable usage of batch statement even if the target database support it.
    • 545238 - Sort tables prior to comparing them. DbUnit now generates an ORDER BY clause in the select statement used by DatabaseDataSet.getTable(). Rows are sorted by primary keys.
    • 554249 - None column values.
    • Added support for BLOB and CLOB data types.
  • Multiple bugfixes:
    • 539089 - FlatXmlDataSet and null values. Now generate metadata from DTD if available. See FlatXmlDataSet documentation.
    • 545234 - Table names in Assertion.assertEquals(IDataSet, IDataSet).
    • 547200 - PreparedStatement.clearParameters() problem with oracle 8i.
    • DeleteAllOperation crash with empty dataset.

Changes in version 1.3 from 1.2.4
  • Added support for binary data types: BINARY, VARBINARY and LONGVARBINARY. Binary data is persisted in xml datasets using the base64 encoding scheme.
  • Added the CaseInsentiveDataSet class.
  • Deprecated the DataSetUtils.assertEquals() methods. You should now use Assertion.assertEquals().
  • Multiple bugfixes:
    • 533321 - Identifiers are case sensitive. See CaseInsentiveDataSet.
    • 533537 - FlatXmlDataSet and null values.
    • 533537 - Problem with single quotes.
    • 534810 - Problem when primary key included all columns

Changes in version 1.2.4 from 1.2
  • Added support for multiple schemas per connection. See How to documentation.
  • Removed element <none/> from dataset.dtd. This feature was incompatible with future support for the binary data types.
  • Fixed bug 530998 - DataSetUtils.AssertEquals(ITable table, ITable table).
  • Fixed bug 533540 - Problem with DATE, TIME and TIMESTAMP.

Changes in version 1.2 from 1.1
  • Added the flat xml dataset format. See the FlatXmlDataSet documentation.
  • Added support for verifying whether two tables or datasets contain identical data. See How to documenation.
  • Improved java API documentation.
  • Updated Electric XML to version 3.2.

Changes in version 1.1 from 1.0
  • Changes that could break older environments:
    • Connections are now closed in each setUp() and tearDown(). You must override the DatabaseTestCase.closeConnection() and provide an empty body to modify this behavior. This is not recommended if you use connection pooling.
    • Modified DatabaseTestCase.getConnection() signature to returns the IDatabaseConnection interface instead of DatabaseConnection. You need to modify your subclasses signature but not necessarily the implementation.
  • Other changes:
    • Added the IDatabaseConnection interface.
    • Added the DatabaseDataSourceConnection class. This class provides support for accessing jdbc connections from a DataSource. Jdbc connections are requested on demand from the DataSource.
    • Improved performance.
    • Fixed bug 526235 - Cannot export tables (jdbc-odbc with MS-SQL 7).
